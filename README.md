# AI Maker Toolkit

A public toolkit of VS Code-first AI skills, agents, workflows, and MCP servers that help me write, publish, present, and build in public while I keep navigating what AI means for my work and career.

I use [my blog](https://thor-draperjr.github.io) as a working library: security, cloud, AI, leadership, career development, and the long game of execution. This repo is the packaging shelf for the AI-assisted workflows that keep showing up in that work.

I do not think of myself as a full-time developer. I think of myself as a non-traditional operator, builder, teacher, and writer who uses VS Code as a creative workspace. AI has made it possible for me to become more of a maker: articles, talks, visual explainers, social posts, small tools, and reusable workflows.

Right now the repo starts with content publishing and social promotion: skills for article visuals and social posts, one voice-and-publish review agent, and one MCP server that adds deterministic social-promotion tools. Future packages should earn their place the same way: small, copyable, and grounded in real work from my blog, talks, projects, and career experiments.

Each package should make three things clear:

| Part | What It Shows |
|---|---|
| What to copy | The exact file or folder someone can adopt in their own VS Code repo. |
| Proof | Screenshots, public links, or a concrete example showing the workflow worked. |
| Why it helps | A short note on the career, writing, learning, or making problem it solves. |

Some packages come from my Astro blog. Others come from social publishing, MCP tooling, career experiments, or whatever repeatable workflow proves useful enough to package. The root README is only the map; the details live with each package.

## Who This Is For

This is for career-minded professionals who use AI to make things: articles, demos, talks, visual explanations, social posts, portfolio pieces, learning projects, and practical tools.

You do not have to identify as a full-time developer for these workflows to be useful. The point is to package the repeatable parts of making, writing, learning, and publishing so an AI agent can help without flattening your voice or turning every project into a software product.

## Browse The Toolkit

This repo has four lanes:

| Lane | Where It Lives | What Belongs There |
|---|---|---|
| Skills | [skills/](skills/) | Copyable capabilities packaged by category and skill name. |
| Agents | [agents/](agents/) | Specialist reviewers or collaborators with a narrower job. |
| MCP servers | [mcp-servers/](mcp-servers/) | Deterministic tool backends that skills can call when available. |
| Workflows | [workflows/](workflows/) | Short playbooks that combine skills, agents, proof, and human gates. |

Each lane README carries the current inventory. Individual package READMEs carry the copy path, examples, screenshots, proof, and why the package is useful.

The [.github/](.github/) folder is different: it only contains the live Copilot instruction, agent, and prompt files this repo uses in VS Code. Public explanation stays in the package folders above.
